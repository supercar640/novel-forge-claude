# Novel Forge Claude v1.7 - 웹소설 작성 도구 구현 계획서

## 개요

PD(기획자)와 AI가 협업하여 웹소설을 기획하고 집필하는 인터랙티브 작성 도구.
PD의 의사결정(선정/보류/폐기/수정)을 중심으로 단계별로 진행되며, 총 4개의 Phase로 구성된다.

---

## Phase 1: 컨텍스트 수립 (방향성 선정 → 기획안 완성)

### 1-1. 방향성 제안

| 항목 | 내용 |
|------|------|
| **입력** | PD가 장르명, 연관 키워드, 분위기 등을 제시 |
| **처리** | AI가 해당 정보를 바탕으로 서로 다른 방향성 5개를 짧은 요약 형태로 생성 |
| **출력** | 번호가 매겨진 방향성 리스트 (각 2~3줄 요약) |

### 1-2. PD 의사결정 (방향성)

| 판정 | 설명 |
|------|------|
| **선정** | 5개 중 **1개만** 선택 가능. 선정 즉시 다음 단계로 이동 |
| **보류(Keep)** | 당장 선택하지 않지만 후보로 유지. 보류된 방향성은 `shelve/idea_*.md`에 자동 저장 (v1.7) |
| **폐기** | 후보에서 제거 |
| **리트라이** | 선정할 만한 항목이 없을 경우, 전체를 폐기하고 새로운 5개를 재생성 |

- 리트라이 시 PD에게 추가 키워드나 방향 힌트를 요청하여 품질 개선

### 1-3. 기획안 빌드업

| 항목 | 내용 |
|------|------|
| **입력** | 선정된 방향성 1개 |
| **처리** | AI가 구체적인 기획안으로 확장 (세계관, 주인공, 컨셉트, 플롯, 테마, 그 외 톤앤매너, 예상 분량 등) |
| **출력** | 구조화된 기획안 문서 |

### 1-4. PD 의사결정 (기획안)

| 판정 | 설명 |
|------|------|
| **선정(승인)** | 기획안 확정 → Phase 1 완료, 컨텍스트 확정 |
| **수정** | PD가 수정 지시 → AI가 반영 후 재제출 (반복 가능) |
| **폐기** | 기획안 폐기 → 1-1 단계로 복귀하여 방향성부터 재시작 |

### 1-5. 컨텍스트 확정

**선정(승인) 시**, AI는 확정된 기획안을 바탕으로 프로젝트 디렉토리의 `context/` 폴더에 아래 **8개의 마크다운 파일**을 생성한다:

| 파일 | 내용 |
|------|------|
| `character_profiles.md` | 주인공, 조연 등 캐릭터 프로필 및 관계도 |
| `setting_world.md` | 시대/공간적 배경 및 고유 규칙 |
| `concept.md` | 로그라인, 장르, 매력 포인트 |
| `plot_outline.md` | 전체 시놉시스와 플롯 뼈대 |
| `themes.md` | 스토리를 관통하는 테마와 메인 상징물 |
| `tone.md` | 톤앤매너, 예상 분량 등 기타사항 |
| `foreshadow.md` | 미회수 복선 목록 (v1.5) |
| `payoff.md` | 회수된 복선 기록 (v1.5) |

또한 `polishing/guideline.md`에 퇴고 가이드라인을 생성한다.

### 1-6. 원고 임포트 (v1.5)

기존 원고가 있는 경우, Phase 1의 방향성 제안 단계에서 `import-manuscript` 명령으로 원고를 임포트할 수 있다.

| 항목 | 내용 |
|------|------|
| **입력** | PD가 기존 원고 파일 경로를 제공 |
| **처리** | AI가 원고를 분석하여 캐릭터, 세계관, 플롯, 테마, 복선 등을 추출하고 컨텍스트 파일(8개)과 퇴고 가이드라인을 생성 |
| **출력** | context/ 폴더에 8개 마크다운 파일 + polishing/guideline.md |

**임포트 플로우:**
1. `import-manuscript <file>` → `import_analysis` 단계로 이동
2. AI가 원고를 분석하고 컨텍스트 파일 생성
3. `save` + `next` → `import_review` 단계로 이동
4. PD가 생성된 컨텍스트를 검토:
   - **승인**: Phase 2(전개 선정)로 직행
   - **수정**: 분석 단계로 돌아가서 수정
   - **폐기**: 방향성 제안(1-1)으로 복귀

### 1-7. 컨텍스트 직접 임포트 (v1.5)

이미 작성된 `context/` 파일이 있는 경우, `import-context` 명령으로 Phase 2로 직행할 수 있다.

| 항목 | 내용 |
|------|------|
| **입력** | context/ 디렉토리에 기존 마크다운 파일이 존재 |
| **처리** | context/ 파일 존재 여부 확인 후 Phase 2로 직행 |
| **출력** | Phase 2(전개 선정) 시작 |

### Phase 1 완료 조건
- PD가 기획안을 **승인**하면 해당 기획안이 **컨텍스트**로 확정되며 Phase 2로 진행
- 또는 `import-manuscript` → 분석 → 승인으로 Phase 2 진행
- 또는 `import-context`로 기존 컨텍스트를 사용하여 Phase 2 직행

## Phase 2: 전개 선정 (스토리 전개 옵션 제시 → 전개 확정)

### 2-1. 전개 옵션 생성

| 항목 | 내용 |
|------|------|
| **입력** | Phase 1에서 확정된 컨텍스트 + `shelve/dev_*.md` 보류 전개 (v2.1) |
| **처리** | AI가 **완전히 다른 방향성**을 가진 5개의 전개 옵션을 생성 |
| **출력** | 각 옵션에 아래 요소를 포함 |

**보류 전개 재활용 규칙 (v2.1):**
- `shelve/dev_*.md` 파일이 있으면 해당 전개를 5개 옵션 중에 **우선 포함**
- 보류된 전개의 probability는 기존 값 유지
- 보류된 전개의 텍스트는 현재 맥락에 맞게 다듬기 가능
- 나머지 슬롯은 새로운 전개로 채움 (3-카테고리 분포 규칙 준수)

**각 전개 옵션의 구성:**

```
옵션 N.
<text>방향성 요약 내용</text>
<probability>0.XX</probability>
```

- `<text>`: 해당 전개의 구체적 방향성 요약
- `<probability>`: 일반적인 소설 데이터베이스 기준 예상 등장 확률 (0.0 ~ 1.0)
  - 1.0에 가까울수록 뻔한 전개
  - 0.0에 가까울수록 독창적이고 기발한 전개

**샘플링 분포 규칙 (v1.5 3-카테고리):**

| 구분 | 개수 | probability 범위 | 설명 |
|------|------|------------------|------|
| 일반(Normal) 옵션 | 2개 | > 0.30 | 자연스럽고 예측 가능한 전개 |
| 중간(Moderate) 옵션 | 1개 | 0.10 ~ 0.30 | 약간 의외이지만 납득 가능한 전개 |
| 희귀(Rare) 옵션 | 2개 | < 0.10 | 매우 독창적이고 기발한 전개 |

- 3-카테고리 시스템으로 전개의 다양성과 균형을 개선
- 희귀 옵션이라도 스토리 맥락에 부합해야 함

### 2-2. PD 의사결정 (전개)

| 판정 | 설명 |
|------|------|
| **선정** | **정확히 1개**만 선택. 선정 즉시 전개 선정 확인 단계로 자동 이동 |
| **보류(Keep)** | 당장 선택하지 않지만 후보로 유지. 보류된 전개는 `shelve/dev_*.md`에 자동 저장 (v1.7) |
| **폐기** | 해당 옵션 제거 |
| **전체 폐기 후 리트라이** | 마음에 드는 옵션이 없으면 5개 전부 폐기 후 재생성 |

### 2-3. 전개 선정 확인

- 전개 선정(1개) 후 AI가 반드시 확인: **"전개 선정을 종료하시겠습니까?"**
- PD가 종료를 승인하면 Phase 2 완료
- 거부하면 전개 선정 단계로 복귀

### Phase 2 완료 조건
- PD가 1개의 전개를 선정하고, 종료를 승인하면 Phase 3으로 진행

## Phase 3: 집필 (소설 작성 → 승인)

### 3-1. 문체 설정

| 항목 | 내용 |
|------|------|
| **처리** | AI가 PD에게 참고할 문체(작가명, 작품명, 문체 설명 등)를 요청 |
| **입력** | PD가 문체 레퍼런스를 제공하거나 "없음"으로 응답 |
| **적용** | 문체가 있으면 해당 스타일을 모방하여 집필, 없으면 AI 기본 문체로 작성 |

### 3-2. 작성 모드 선택

PD가 아래 3가지 작성 모드를 조합하여 선택:

| 모드 | 설명 | 분량 |
|------|------|------|
| **자동작성(auto)** | AI가 3화를 자율 연쓰기 (전개 생성→카테고리 선택→전개 선택→집필→퇴고 내부 반복) | 3화 × 5,500자 이상 |
| **장면별(scene)** | PD 확인하며 장면별 작성. 장면마다 PD 피드백 루프 (승인/수정/폐기). PD가 `merge-episode`로 수동 병합하여 에피소드 완성 | 장면 단위 (병합 시 5,500자 이상) |
| **1화 분량(episode)** | PD 확인하며 1화 분량 일괄 작성 | 5,500자 이상 |

- auto와 scene/episode는 상호 배타적으로 선택 (동시 선택 불가) (v1.7)
- scene과 episode는 동시 선택 불가
- 최소 하나의 모드는 설정되어야 진행 가능
- **`switch-auto` (v1.5)**: 모드 선택 또는 집필 도중에 auto 모드로 전환 가능

### 3-3. 장면별 모드 (scene) 상세

scene 모드에서는 장면 단위로 작성→검토를 반복한 뒤, PD가 수동으로 병합하여 에피소드를 완성한다.

**장면 작성 흐름:**

```
WRITING → save(sc001.md) → next → SCENE_DECISION
  → approve  → scene_count++ → WRITING (다음 장면)
  → revise   → 마지막 장면 제거 → WRITING (현재 장면 수정)
  → reject   → 마지막 장면 제거 → WRITING (현재 장면 재작성)
  → merge-episode → 5,500자 검증 → 병합 → WRITING_DECISION
    → approve  → Phase 4 (퇴고)
    → revise   → scene_count=0, WRITING (처음부터)
    → reject   → scene_count=0, WRITING (처음부터)
```

**핵심 규칙:**
- 장면 파일은 `sc001.md`, `sc002.md`, ... 네이밍 (episode의 `ep_draft.md`와 구분)
- `scene_count` 필드로 승인된 장면 수를 추적 (episode_count와 독립)
- `scenes` 명령으로 승인된 장면 목록과 글자 수를 확인:
  ```
  === 장면 목록 ===
    1. sc001.md (1,234자)
    2. sc002.md (1,987자)
    ────────────────────
    누적: 3,221자 / 5,500자 기준
  ```
- `merge-episode` 명령으로 장면들을 하나의 에피소드 초안(`ep_draft.md`)으로 병합
- **병합 게이트**: 누적 글자 수가 5,500자 미만이면 병합이 차단됨. PD는 장면을 더 작성해야 함
- 병합 후 `WRITING_DECISION` 단계에서 에피소드 전체를 검토

**PD 의사결정 (장면 검토, `scene_decision`):**

| 판정 | 설명 |
|------|------|
| **승인** | 장면 승인 → scene_count 증가, 다음 장면 작성으로 이동 |
| **수정** | 마지막 장면 제거 후 수정 재작성 (기존 장면은 보존) |
| **폐기** | 마지막 장면 제거 후 재작성 (기존 장면은 보존) |
| **merge-episode** | 승인된 장면들을 하나의 에피소드로 병합 (5,500자 이상 필요) |
| **scenes** | 승인된 장면 목록과 글자 수 확인 (상태 변경 없음) |

### 3-4. PD 의사결정 (원고)

| 판정 | 설명 |
|------|------|
| **승인** | 원고 확정 → Phase 3 완료 |
| **보류(Hold)** | 현재 초안을 `shelve/draft_*.md`에 저장하고 집필(writing) 단계로 복귀하여 새로 작성 (v1.7) |
| **수정** | PD가 수정 지시 → AI가 반영 후 재제출 (반복 가능) |
| **폐기** | 원고 폐기 → 3-3 단계로 복귀하여 재작성 |
| **PD 퇴고 (v1.5)** | `pd-proofread <file>`: PD가 직접 퇴고한 원고를 등록하면 AI 퇴고를 건너뛰고 컨텍스트 갱신으로 직행 |

### Phase 3 완료 조건
- PD가 원고를 **승인**하면 해당 회차 집필 완료 → Phase 4로 진행
- 또는 PD가 `pd-proofread`로 직접 퇴고한 경우 Phase 4 퇴고 단계를 건너뛰고 컨텍스트 갱신으로 직행

## Phase 4: 퇴고 및 컨텍스트 갱신 (원고 확정 → 컨텍스트 업데이트 → Phase 2 복귀)

### 4-1. 퇴고 (교정·교열)

| 항목 | 내용 |
|------|------|
| **입력** | Phase 3에서 승인된 원고 + 현재 컨텍스트 + 퇴고 가이드라인(`polishing/guideline.md`) |
| **처리** | AI가 원고를 재검토하여 문체 일관성, 오탈자, 문맥 흐름, 캐릭터 행동 일관성, 설정 충돌 등을 점검 |
| **출력** | 수정 사항이 표시된 퇴고 결과물 |

- **PD 퇴고 (v1.5)**: 퇴고 단계에서도 `pd-proofread <file>`로 PD가 직접 퇴고한 원고를 등록하면 컨텍스트 갱신으로 직행 가능
- **퇴고 가이드라인**: `polishing/guideline.md`에 작품별 퇴고 기준을 정의하여 일관된 퇴고 수행

### 4-2. PD 의사결정 (퇴고)

| 판정 | 설명 |
|------|------|
| **승인** | 퇴고 완료 → 원고 최종 확정, 4-3 컨텍스트 갱신으로 이동 |
| **수정** | PD가 추가 수정 지시 → AI가 반영 후 재제출 (반복 가능) |
| **폐기** | 퇴고 결과 폐기 → Phase 3으로 복귀하여 재집필 |

### 4-3. 컨텍스트 갱신

| 항목 | 내용 |
|------|------|
| **입력** | 최종 확정된 원고 + 기존 컨텍스트 파일들 |
| **처리** | AI가 확정 원고에서 발생한 변경 사항을 분석하여 컨텍스트 파일 업데이트 |
| **대상** | `context/` 폴더 내 해당하는 파일들 (8개) |

**갱신 항목:**
- 새로 등장한 캐릭터 또는 기존 캐릭터의 관계 변화 → `character_profiles.md`
- 새로 밝혀진 세계관 요소 또는 설정 변화 → `setting_world.md`
- 플롯 진행 상황 반영 → `plot_outline.md`
- 테마·복선 전개 추적 → `themes.md`
- **복선 관리 (v1.5)**:
  - 새로 설치된 복선 → `foreshadow.md`에 추가
  - 회수된 복선 → `foreshadow.md`에서 제거하고 `payoff.md`에 기록

### 4-4. 컨텍스트 크기 관리

AI가 갱신 후 컨텍스트 전체 크기를 평가하여, AI 처리에 부담이 되는 수준이라고 판단될 경우 자동으로 압축 프로세스를 실행한다.

**압축 프로세스:**

| 단계 | 설명 |
|------|------|
| **1. 백업 생성** | `backup/context_v{N}/` 폴더를 생성하고, 현재 `context/` 파일들을 전부 복사 (N은 자동 증가 버전 번호) |
| **2. 요약본 재생성** | 기존 컨텍스트의 핵심 내용만 추출하여 압축된 요약본으로 `context/` 파일들을 교체 |
| **3. PD 확인** | AI가 PD에게 압축 사실과 요약본을 제시하고, PD가 승인하면 확정. 수정 요청 시 요약본 조정 |

### 4-5. Phase 2 복귀

- 컨텍스트 갱신(및 필요 시 압축)이 완료되면, **갱신된 컨텍스트를 기반으로 Phase 2로 복귀**하여 다음 회차 전개 선정을 시작
- 이 순환(Phase 2 → 3 → 4 → 2)이 연재가 끝날 때까지 반복

### Phase 4 완료 조건
- PD가 퇴고를 승인하고, 컨텍스트 갱신(필요 시 압축 포함)이 완료되면 Phase 2로 복귀

## 언제든 퇴고 모드 (v2.1)

이미 완성된 에피소드를 언제든지 다시 수정할 수 있는 모드. `revise-episode <file>` 명령으로 진입한다.

### 진입 가능 단계

v2.1부터 **대부분의 단계에서 퇴고 모드 진입이 가능**하다. 단, 다음 단계에서는 진입 불가:

| 진입 불가 단계 | 이유 |
|---------------|------|
| `direction_decision` | 방향성 선택 중 |
| `plan_buildup` | 기획안 작성 중 |
| `writing` | 집필 진행 중 |
| `proofreading` | 퇴고 진행 중 |

그 외 모든 단계에서 퇴고 모드 진입 가능 (단, episode_count > 0이어야 함).

### 워크플로우

```
revise-episode <file>
  → drafts/revision_<file> 생성 (원본 복사)
  → proofread_decision 단계로 직접 진입
  → AI가 퇴고본 작성 (drafts/revision_<file> 수정)
  → PD 의사결정:
    → approve → context_update 단계로 이동
    → revise "<피드백>" → proofreading 단계에서 재퇴고
    → reject → 퇴고 취소, 원래 단계로 복귀 (변경 없음)
  → 컨텍스트 갱신 후 next → 원본 에피소드 덮어쓰기
  → 원래 진입했던 단계로 복귀
```

### 핵심 규칙

- **episode_count 불변**: 재수정은 기존 회차를 덮어쓰는 것이므로 episode_count가 증가하지 않음
- **원본 덮어쓰기**: 컨텍스트 갱신 완료 후 수정된 원고가 원본 에피소드 파일(`episodes/` 내)을 직접 대체
- **컨텍스트 갱신**: 수정된 내용에 따라 Phase 4 컨텍스트 갱신을 정상적으로 수행
- **복귀**: 퇴고 완료 후 원래 진입했던 단계로 자동 복귀
- **단순화된 흐름**: v2.1에서는 context_update → 바로 복귀 (context_size_check, complete 단계 생략)

## 구현 시 핵심 원칙

1. **PD 중심 의사결정**: 모든 단계에서 AI는 제안하고, PD가 최종 판단
2. **반복 가능한 루프**: 각 단계에서 만족스러운 결과가 나올 때까지 리트라이 가능
3. **순환 구조**: Phase 2 → 3 → 4 → 2 사이클을 반복하며 연재를 이어감
4. **상태 관리**: 선정/보류/폐기 상태를 명확히 추적
5. **3-카테고리 확률 기반 창의성 조절**: Phase 2에서 일반/중간/희귀 전개를 분리하여 다양성 보장 (v1.5)
6. **모드 배타성**: auto와 scene/episode는 상호 배타적으로 선택 (v1.7)
7. **컨텍스트 지속 관리**: 회차 진행에 따라 컨텍스트를 갱신하고, 크기 부담 시 백업 후 요약 압축
8. **복선 추적**: foreshadow.md/payoff.md로 복선 설치·회수를 체계적으로 관리 (v1.5)
9. **유연한 진입 경로**: 신규 기획 외에도 기존 원고 임포트, 기존 컨텍스트 임포트로 빠르게 시작 가능 (v1.5)
10. **인코딩 무결성**: UTF-8 인코딩 검증으로 컨텍스트 파일 손상 감지 (v1.5)
11. **초안 중복 방지**: draft_files에 동일 파일 중복 등록 방지 (v1.5)
12. **장면별 모드**: scene 모드에서 `scene_count`로 장면 수 추적, `merge-episode`로 수동 병합 (5,500자 게이트), `scenes`로 장면 목록 확인
13. **언제든 퇴고**: `revise-episode <file>`로 대부분의 단계에서 퇴고 모드 진입 가능, episode_count 불변 (v2.1)
14. **보류 전개 재활용**: 보류(Hold)된 전개는 `shelve/dev_*.md`에 저장되고, 다음 전개 제안 시 5개 옵션에 우선 포함 (v2.1)

| `hold <id>` | 항목 보류 |
| `discard <id>` | 항목 폐기 |
| `retry` | 전체 폐기 후 재생성 요청 |
| `approve` | 현재 단계 승인 |
| `revise "<feedback>"` | 수정 요청 (피드백 포함) |
| `reject` | 폐기하고 이전 단계로 복귀 |
| `confirm-end` | 전개 선정 종료 확인 (Phase 2 전용) |
| `save <type> <file>` | 초안 파일 저장 (plan/manuscript/proofread) |

| `next` | 다음 단계로 진행 |

| `revise-episode <file>` | 언제든 퇴고 모드 진입 - 완성된 에피소드 재수정 (v2.1) |
